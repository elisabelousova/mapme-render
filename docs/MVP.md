# MVP: бот из сохраненок в карту

## Цель

Проверить, правда ли людям удобно отправлять travel-контент в Telegram-бота и получать из этого личную карту по городам.

## Главное пользовательское действие

Пользователь отправляет боту ссылку, текст, подпись к Reels или список мест. Бот отвечает найденными местами и дает ссылку на личную карту.

## MVP-сценарий

1. Пользователь запускает бота.
2. Отправляет текст или несколько строк с местами.
3. Бот выделяет кандидатов.
4. Бот сохраняет их в личную библиотеку.
5. Пользователь открывает веб-страницу.
6. Видит города, места, категории и ссылки на Google Maps.

## Почему Telegram, а не Instagram на первом шаге

Telegram проще для технического MVP:

- бот может принимать текст, ссылки, фото и документы;
- не нужно проходить Meta App Review до проверки гипотезы;
- можно быстро дать пользователю личную ссылку на карту;
- аудитория уже приходит в Telegram-канал.

Instagram можно добавить позже как канал ввода, если подтвердится ценность продукта.

## Границы MVP

Входит:

- Telegram-бот;
- хранение мест;
- личная карта-список;
- ручное добавление;
- Google Maps search links.

Не входит:

- автоматический доступ к Instagram Saved;
- гарантированное чтение закрытых Reels;
- полноценная оптимизация маршрута;
- платные тарифы;
- совместные карты.

## Метрики проверки

- сколько людей отправило хотя бы 1 место;
- сколько добавило 5+ мест;
- сколько открыло карту после ответа бота;
- сколько вернулось к карте через сутки;
- сколько мест пришлось исправлять вручную.

## После MVP

1. AI-распознавание текста и скринов.
2. Places API для проверки реальных адресов.
3. Сбор маршрута на день.
4. Экспорт в Google Maps.
5. Платный лимит после 20-30 мест.
